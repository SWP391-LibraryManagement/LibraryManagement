# Project Overview

## 1. Project Name

**Library Management System**

## 2. Problem Statement

Many libraries still manage books, members, borrowing records, returns, reservations, fines, and study seat reservations manually or with separated tools. This can lead to duplicated records, difficulty in tracking book availability, delayed return management, inaccurate fine calculation, and conflicts in seat booking schedules.

The Library Management System is developed to help the library manage books, book copies, members, borrowing and returning activities, book reservations, fines, study seat reservations, and basic reports in a centralized system.

## 3. Project Objectives

The main objectives of the Library Management System are:

- To manage book information and physical book copies.
- To support members in searching, borrowing, reserving, and renewing books.
- To support members in reserving study seats in library areas.
- To help librarians process book borrowing, returning, reservations, fines, and seat booking requests.
- To allow administrators to manage users, roles, loan policies, seat booking policies, and reports.
- To reduce manual work and improve the accuracy of library operations.
- To provide basic notifications for due dates, overdue books, book reservation status, and seat booking status.

## 4. Target Users

The system serves the following user groups:

| User Group           | Description                                                                                                                              |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Guest                | A visitor who can search and view public book information and register for an account.                                                   |
| Member               | A registered library user who can borrow books, reserve books, renew loans, view borrowing history, view fines, and reserve study seats. |
| Librarian            | A staff member who manages books, book copies, members, loans, returns, reservations, fines, and seat booking requests.                  |
| Admin                | A system administrator who manages users, roles, loan policies, seat booking policies, reports, and audit logs.                          |
| Notification Service | An external service used to send due date, overdue, book reservation, and seat booking notifications.                                    |

## 5. Main Features

The main features of the system include:

- Authentication and role management
- User and member management
- Book management
- Book category and author management
- Book copy management
- Search, filter, and browse books
- Borrow book
- Return book
- Renew loan
- Borrowing history management
- Book reservation and reservation queue
- Study seat reservation
- Fine management for overdue, lost, and damaged books
- Basic notification
- Basic reports and data export

## 6. Expected Outcome

After completion, the system is expected to provide a centralized platform for managing library operations. Guests can search for books, members can manage their borrowing activities and reserve study seats, librarians can handle daily library transactions and seat booking requests, and administrators can monitor system data and basic reports.

The system should improve the efficiency, accuracy, and traceability of book borrowing, returning, book reservation, fine management, and study seat reservation processes.
